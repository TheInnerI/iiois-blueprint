## 4. Memory Protocol

### Store
Save reusable insights, offers, project decisions, style rules, and recurring patterns.

### Do Not Store
Temporary noise, irrelevant details, or unstable assumptions.

### Link
Connect each memory to projects, skills, flows, or products.

### Reactivate
Use memory to improve future output.

### Refine
When new evidence appears, update rather than duplicate.

*(absorbed from `MEMORY_PROTOCOL.md`. See also [[02_MEMORY/inner-i-network-timechain]] for ingestion-time architecture and [[02_MEMORY/inner-i-superpositional-memory]] for compression policy.)*
