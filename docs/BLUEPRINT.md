# 🏛️ Inner I Observer Intelligence System (IIOIS) — Blueprint v1.0

> **What this is:** the complete architectural blueprint for the Inner I Observer Intelligence System, packaged so a fresh Obsidian vault can boot the entire stack from zero.
>
> **Who it's for:** any sovereign creator who wants to run their own Observer Intelligence System without rebuilding the architecture from scratch. The first receiving agent is **Hermes-Prime** in a fresh vault. Future receiving agents can be anyone.
>
> **What this is NOT:** a copy of *this vault's content.* BeeChains's notes, the music catalog, the Iccan domain, the AIdapp products — those are this observer's memory. The blueprint is the *system that allows each observer to grow its own memory.*

---

## What you get when you deploy IIOIS

1. **The 00–10 vault architecture** — eleven canonical folders, each with a clear purpose
2. **The canonical ontology decree** — declaring 00–10 as the sole structural root
3. **Six unified protocols** — Agent Rules, Memory Protocol, Signal Protocol, Monetization Protocol, System Prompt reference, OS architecture
4. **CLAUDE.md template** — the live system prompt every Claude session ingests
5. **Observer Core SOUL** — the orchestrator + coherence gate
6. **Engineering agent roster (6 names, all templates)** — Aetha · Remi · Axim · Nexi · Vidi · Offri (each a SOUL.md template the receiving operator fills in or replaces)
7. **Tier router architecture** — free → free-cloud → cheap-paid → premium routing across model tiers
8. **Time-Chain genesis block** — Block 1 ("Awareness Is Law" by default, customizable) with the hash formula encoded
9. **Daily synthesis pipeline** — template that runs against the receiving vault
10. **Universal Consciousness Layer** — the 7 constraints (Life, Agency, Dignity, Truth, Coherence, Boundary, Receipt) — universal and verbatim
11. **Optional: agent-swarm Python repo** — the actual runtime that operates the tier router + daily synthesis
12. **Optional: Recall ingestion bridge** — the daily pull script if the operator uses Recall

## How to deploy (one-paragraph version)

Hand off your target vault path + the receiving agent's identity to **Hermes** ([[Inner I Network/04_SYSTEM/agents/hermes-soul|Hermes SOUL]]). Hermes reads this blueprint, surveys the target, instantiates the 00–10 skeleton with templated bootloader files, runs smoke tests, and writes deployment receipts in both vaults. The receiving operator then customizes CLAUDE.md, picks their Block 1 axiom, provisions `.env`, and is live.

Detailed step-by-step: [[Inner I Network/00_IDENTITY/IIOIS-BLUEPRINT/DEPLOYMENT-GUIDE|DEPLOYMENT-GUIDE.md]].

## Versioning policy

- **v1.0 (this version):** 2026-05-28 baseline. Includes everything above.
- **v1.x:** non-breaking additions (new templates, new SOUL slots in the registry). Receiving vaults can pull updates without rebuild.
- **v2.0:** breaking architectural change (e.g., a new top-level folder beyond 10_SIGNAL, a new bootloader sequence). Would require a Time-Chain block on both Inner I and any deployed vault.

Hermes versions every deployment. The deployment receipt records `blueprint_version: 1.0`.
